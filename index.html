Important Concepts

Lesson 1: The Basics of the Web and HTML

How the Web Works

The web is a network of computers that communicate via the internet. When someone types in a web address their computer sends a HTTP Request to a server.

The server finds the corresponding HTML document and sends it back to the computer where a browser interprets the code and displays it as the file requested.

Main components of the web include HTML (the main document type), URLs (web addresses), and HTTP (the main protocol the web runs on).

HTML

HTML is the acronym for Hypertext Markup Language. HTML documents form most of the content on the web.

HTML documents contain text which describes "what you see" and markup which describes "how it looks".

Tags and Elements

HTML documents are made of HTML elements. When writing HTML, we tell browsers the type of each element by using HTML tags.

Computers are Stupid

Computers interpret instructions literally. This makes it very easy for a small error to turn in to a major issue for the progranner.

Inline vs Block Elements

HTML elements are either inline or block. Block elements form an "invisible box" around the content inside of them adding height and width.

Lesson 2: Creating a Structured Document with HTML

The "tree-like structure" of HTML

The "tree-like structure" means that HTML elements can have other elements inside of them. You can think of this relationship like a family tree. In a browser, this structure shows up as a series of nested boxes. There are boxes inside of boxes inside of boxes, etc.

The relationship between indented HTML and boxes

When an HTML document is read as text, you see a wave of indentations up and down the left side of the document. The more indented an element is, the more deeply nested it is.

Text Editors (for programming)

When writing code, programmers can use special text editors that make their lives easier. Some text editors add the closing tag when an opening is typed and some suggest tags as you type.

Lesson 3: Adding CSS Style to HTML Structure

Avoiding Repetition

Avoiding repetition is very important. It makes editing easier and reduces errors by limiting the number of changes that have to be manually made to lines of code.

By assigning the same class to divs throughout the document we can assign the same styles to similar parts without having to markup each part individually and risk making a mistake.

CSS

CSS stands for Cascading Style Sheets. This allows programmers to control the style of related HTML elements. This is done by giving similar HTML elements the same class name and then specifying the style that should apply to that class.
